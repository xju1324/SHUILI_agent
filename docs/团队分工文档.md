## Git 协作规范

1. **分支策略**: 每人从 `main` 分支创建自己的功能分支（`feature/backend`, `feature/frontend`, `feature/ai-service`, `feature/docs`）
2. **提交规范**: 使用中文提交信息，格式 `[模块] 简要说明`，如 `[后端] 新增材料CRUD接口`
3. **合并流程**: 功能完成后提交 Pull Request，至少一人 Review 后合并
4. **提交数量**: 每人至少5次有意义的提交（非合并提交）
