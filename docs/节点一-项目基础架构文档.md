### 核心表说明

**users (用户表)**: 存储系统用户信息，角色分为 ADMIN/REVIEWER/APPLICANT

**materials (材料表)**: 取水许可申请材料主表，包含：
- 业务字段: 项目名称、取水地点、取水用途、年取水量等
- 文件字段: 申请表、营业执照、身份证件、水资源论证报告等附件路径
- 状态字段: DRAFT(草稿) → SUBMITTED(已提交) → REVIEWING(审查中) → APPROVED(已通过) / REJECTED(未通过)

**review_records (审查记录表)**: AI审查结果，支持三种审查类型：
- FORMAT: 形式审查（材料完整性）
- DATA_STANDARD: 数据规范性审查
- COMPLIANCE: 实质合规审查

**knowledge_documents (知识库文档表)**: 管理上传到知识库的法规、标准文档

## 四、API 接口设计

### Java 后端 RESTful API (端口 8080)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/materials | 获取材料列表 |
| GET | /api/materials/{id} | 获取材料详情 |

|------|------|------|

## 六、项目启动说明

### 环境要求

- JDK 17+
- Maven 3.8+
- Python 3.9+
- Node.js 18+
- MySQL 8.0

### 启动步骤

```bash
# 1. 克隆仓库
git clone <your-github-repo-url>
cd water-permit-review

# 2. 初始化数据库
mysql -u root -p < backend/sql/schema.sql

# 3. 修改 backend/src/main/resources/application.yml 中的数据库密码

# 4. 启动 Java 后端 (终端1)
cd backend
mvn spring-boot:run

# 5. 启动 Python AI 服务 (终端2)
cd ai-service
pip install -r requirements.txt
python main.py

# 6. 启动前端 (终端3)
cd frontend
npm install
npm run dev
```

### 验证

- Python AI: http://localhost:8000/docs
- 前端: http://localhost:3000
